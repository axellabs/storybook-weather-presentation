Testing React Components with Storybook.js

by Axel Simms

What Is Storybook.js?

a UI development tool for designing and developing components in an isolated environment (outside of your app)

Helpful Features

  • Act as a living style guide: a source of truth for components that updates with your code
  • Test different cases without manipulating your entire app
  • Visually inspect components across browsers + operating systems in one place

Development Processes

Test Driven Development (TDD)

Converting requirements into tests and ensuring that those tests pass

Behavior Driven Development (BDD)

an evolution of TDD, still relies on software passing certain tests, but testing is focused on the end behavior of the system from the users' perspective

Component Driven Development (CDD)

  • development is focused on building components and composing them into larger structures
  • Influenced by Atomic Design (Brad Frost)

Storybook Driven Development (SDD)

  • Also known as style guide driven development
  • A mix of CDD and TDD, writing stories as tests and visually inspecting as we go, while preventing regressions with automated visual testing
